# Changelog

本專案的所有顯著變更將記錄在此檔案中。
格式參考自 [Keep a Changelog](https://keepachangelog.com/zh-TW/1.1.0/)。

## [1.3.0] - 2026-04

### Added

- **經驗值獲得途徑擴充**：
    - 現在除了文字訊息，發送媒體（圖片、影片、檔案）與貼圖也能獲得經驗值 (XP)。
    - 優化訊息過濾邏輯，自動排除系統訊息（如成員加入、置頂等）以維持公平性。

### Changed

- **訊息處理機制優化**：
    - 調整 `MessageHandler` 的過濾器，從 `filters.TEXT` 變更為複合式過濾，支援更多類型的互動。
    - 強化 `thread_id` 識別邏輯，確保在論壇 Topic 中的回覆更為精準。

## [1.2.0] - 2026-04

### Added

- **指令選單系統**：
    - 新增 `BotCommand` 支援，機器人啟動後會自動設定 Telegram 指令選單（`post_init` 鉤子）。
    - 使用者現在可以透過輸入框旁的選單直接選取指令。

### Changed

- **程式碼最佳化**：
    - 重構管理員與權限檢查邏輯，程式碼更為精簡。
    - 指令處理流程優化，提高反應速度與穩定性。
- **指令調整**：
    - 移除 `/level` 指令別名，統一使用 `/rank`。
    - 強化 `/addadmin` 與 `/deladmin` 的 ID 識別邏輯。

## [1.1.0] - 2026-03

### Added

- **管理員系統**：

    - 新增 `admins` 資料表，支援多管理員權限管理。
    - 支援透過 `OWNER_ID` 設定初始擁有者。
    - 新增 `/addadmin` 與 `/deladmin` 指令，動態管理管理員名單。
- **經驗值手動調整**：
    - 新增 `/addxp` 指令，管理員可為指定使用者增加經驗值，支援自動跨多級計算。
- **使用者識別優化**：
    - 資料庫 `users` 表新增 `username` 欄位，支援透過 `@username` 進行管理操作。
- **系統日誌**：
    - 整合 Python `logging` 模組，提供更清晰的機器人運行狀態追蹤。

### Changed

- **進度條邏輯**：優化百分比與進度條長度計算，確保顯示穩定性。
- **程式碼結構**：優化資料庫初始化流程，並統一採用 `telegram.constants.ParseMode`。

## [1.0.0] - 2026-03

### Added

- **專案初始版本發佈**：實現 Telegram 聊天等級系統核心功能。
- **核心經驗值系統**：
    - 每則有效訊息可獲得 1 XP（排除機器人與指令）。
    - 採用標準 RPG 二次模型計算等級門檻（`5L^2 + 50L + 10`）。
- **訊息冷卻機制**：內建 `COOLDOWN_SECONDS` 防止使用者透過短時間內大量發訊來刷等級。
- **等級查詢指令**：
    - 新增 `/rank` 指令：產出包含等級、經驗進度與視覺化進度條的報告。
    - 新增 `/level` 指令：作為 `/rank` 的別名。
- **視覺化進度條**：使用 Unicode 字元生成動態進度條（如 `████░░░░░░`）顯示升級百分比。
- **SQLite 資料庫整合**：自動建立並維護 `chat_levels.db`，跨工作階段儲存使用者等級資料。
- **Telegram Topic 支援**：支援在啟用論壇功能的群組中使用，機器人會自動識別並回覆於正確的 Topic 內。
- **HTML 格式支援**：等級報告與升級通知均採用 HTML 解析模式，支援加粗與 Mention 功能。
