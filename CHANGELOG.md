# Changelog

本專案的所有顯著變更將記錄在此檔案中。
格式參考自 [Keep a Changelog](https://keepachangelog.com/zh-TW/1.1.0/)。

## [2.1.1] - 2026-04

### Fixed

- **修復 JobQueue 崩潰問題**：
    - 解決了因未安裝 `apscheduler` 導致 `context.job_queue` 為 `None` 並引發 `AttributeError` 的問題。
    - 在 `schedule_deletion` 函數中加入防護邏輯，確保在環境未備妥時不會導致程式中斷。

### Changed

- **更新依賴需求**：
    - 將 `requirements.txt` 中的 `python-telegram-bot` 變更為 `python-telegram-bot[job-queue]`。

## [2.1.0] - 2026-04

### Added

- **訊息自動刪除系統**：
    - 新增 `AUTO_DELETE_SECONDS` 配置（預設 60 秒），機器人的指令回應與升級公告會自動定時刪除，保持群組整潔。
- **管理指令回覆支援**：
    - `/addxp` 與 `/addrank` 現在支援「回覆訊息」操作，管理員只需回覆該用戶的訊息並輸入數值即可直接調整。

### Changed

- **多群組數據隔離優化**：
    - 修復管理員指令在多群組環境下可能導致的數據誤更動問題，所有操作現在嚴格鎖定於當前 `chat_id`。
- **搜尋邏輯強化**：
    - 確保管理操作精確對應到目前群組中的使用者數據。
- **UI/UX 優化**：
    - 統一管理員指令的回應格式，並整合自動刪除邏輯。

## [2.0.0] - 2026-04

### Added

- **群組排行榜分頁功能**：
    - 新增 `/top` 指令，支援分頁顯示群組等級排行榜（每頁 10 人）。
    - 使用 `InlineKeyboardButton` 與 `InlineKeyboardMarkup` 提供互動式的上一頁/下一頁導覽。
- **資料庫欄位擴充**：
    - `users` 表新增 `display_name` 欄位，記錄使用者名稱以提升排行榜的可讀性。
- **穩定性強化**：
    - 引入 `HTTPXRequest` 自訂網路請求設定，調高 `connect_timeout` 與 `read_timeout` 至 20.0 秒，有效解決高負載或網路不穩定的 `TimedOut` 報錯。

### Changed

- **安全性與配置優化**：
    - 改用 `.env` 檔案管理機器人 Token，不再硬編碼於主程式中，並整合 `python-dotenv` 模組。
- **查詢邏輯優化**：
    - 調整管理員指令的搜尋邏輯，支援不分大小寫的 Username 查詢。

## [1.5.0] - 2026-04

### Changed

- **資料庫效能大幅優化**：
    - 引入全域資料庫連接 (`db_conn`)，減少頻繁開啟與關閉檔案產生的效能損耗。
    - 啟用 **WAL (Write-Ahead Logging)** 模式，顯著提升並發讀寫效能。
    - 調整同步模式為 `NORMAL`，在確保資料安全的前提下加快寫入速度。
    - 支援非同步環境下的長連接 (`check_same_thread=False`)。
- **資源管理強化**：
    - 新增 `finally` 區塊，確保機器人停止時能安全關閉資料庫連接，避免資料損毀。

### Added

- **系統狀態追蹤**：啟動時會顯示資料庫模式與連接狀態。

## [1.4.0] - 2026-04

### Added

- **等級直接修改功能**：
    - 新增 `/addrank` 指令，管理員可直接增加或減少指定用戶的等級。
- **完善的數值扣除邏輯**：
    - `/addxp` 現在支援負值（減少經驗），並會根據剩餘經驗自動處理降級邏輯。
    - 增加邊界檢查，確保等級與經驗值最低為 0，不會出現負數資料。

### Changed

- **指令選單更新**：同步更新 `BotCommand` 選單，加入新指令 `addrank`。
- **管理邏輯優化**：強化管理員調整數值時的資料庫一致性。

## [1.3.0] - 2026-04

### Added

- **經驗值獲得途徑擴充**：
    - 現在除了文字訊息，發送媒體（圖片、影片、檔案）與貼圖也能獲得經驗值 (XP)。
    - 優化訊息過濾邏輯，自動排除系統訊息（如成員加入、置頂等）以維持公平性。

### Changed

- **訊息處理機制優化**：
    - 調整 `MessageHandler` 的過濾器，從 `filters.TEXT` 變更為複合式過濾，支援更多類型的互動。
    - 強化 `thread_id` 識別邏輯，確保在論壇 Topic 中的回覆更為精準。

## [1.2.0] - 2026-04

### Added

- **指令選單系統**：
    - 新增 `BotCommand` 支援，機器人啟動後會自動設定 Telegram 指令選單（`post_init` 鉤子）。
    - 使用者現在可以透過輸入框旁的選單直接選取指令。

### Changed

- **程式碼最佳化**：
    - 重構管理員與權限檢查邏輯，程式碼更為精簡。
    - 指令處理流程優化，提高反應速度與穩定性。
- **指令調整**：
    - 移除 `/level` 指令別名，統一使用 `/rank`。
    - 強化 `/addadmin` 與 `/deladmin` 的 ID 識別邏輯。

## [1.1.0] - 2026-03

### Added

- **管理員系統**：

    - 新增 `admins` 資料表，支援多管理員權限管理。
    - 支援透過 `OWNER_ID` 設定初始擁有者。
    - 新增 `/addadmin` 與 `/deladmin` 指令，動態管理管理員名單。
- **經驗值手動調整**：
    - 新增 `/addxp` 指令，管理員可為指定使用者增加經驗值，支援自動跨多級計算。
- **使用者識別優化**：
    - 資料庫 `users` 表新增 `username` 欄位，支援透過 `@username` 進行管理操作。
- **系統日誌**：
    - 整合 Python `logging` 模組，提供更清晰的機器人運行狀態追蹤。

### Changed

- **進度條邏輯**：優化百分比與進度條長度計算，確保顯示穩定性。
- **程式碼結構**：優化資料庫初始化流程，並統一採用 `telegram.constants.ParseMode`。

## [1.0.0] - 2026-03

### Added

- **專案初始版本發佈**：實現 Telegram 聊天等級系統核心功能。
- **核心經驗值系統**：
    - 每則有效訊息可獲得 1 XP（排除機器人與指令）。
    - 採用標準 RPG 二次模型計算等級門檻（`5L^2 + 50L + 10`）。
- **訊息冷卻機制**：內建 `COOLDOWN_SECONDS` 防止使用者透過短時間內大量發訊來刷等級。
- **等級查詢指令**：
    - 新增 `/rank` 指令：產出包含等級、經驗進度與視覺化進度條的報告。
    - 新增 `/level` 指令：作為 `/rank` 的別名。
- **視覺化進度條**：使用 Unicode 字元生成動態進度條（如 `████░░░░░░`）顯示升級百分比。
- **SQLite 資料庫整合**：自動建立並維護 `chat_levels.db`，跨工作階段儲存使用者等級資料。
- **Telegram Topic 支援**：支援在啟用論壇功能的群組中使用，機器人會自動識別並回覆於正確的 Topic 內。
- **HTML 格式支援**：等級報告與升級通知均採用 HTML 解析模式，支援加粗與 Mention 功能。
